CarChip Updater, Basic, v 1.51

Note: This firmware is for the CarChip Basic ( #8210 ) only.

What's new in version v1.51
1. VPW protocol improved to address Flashing dash lights/Gauge Quitting problem in Pontiac Grand Prix and other GM/Chevy SUVs.
2. ISO/KWP protocol implementation is improved for bus contention.
3. CarChip now waits 4 seconds upon engine start before sending the first request.
4. Previous known protocol is attempted 3 times upon Engine start, if unsuccessful tries all the protocols.

To update your CarChip:
Simply connect your CarChip to your PC with the serial interface provided with CarChip and run this program (CarChip_Updater_Basic_v151.exe). It will automatically update the firmware in your CarChip.

This firmware update requires purchase of a CarChip Basic (#8210 only), a PC running Windows 95/98/NT/2000/ME/XP and a serial port to connect to the CarChip.

CarChip Software v2.2.0

What's new in version 2.2.0
1. Support for 1 second speed sampling (Enabled in CarChip firmware 2.18 or later).
2. New Tools Menu.
3. Fuel Entry is now under the new tools menu.
4. Support for adding/removing and viewing fuel entries.
5. Support for viewing OBD Readiness Monitors.
6. Support for Trip Type Entry. Allows users to select whether a trip was for business/personal/commute/other.
7. Accumulates distance traveled between certain dates for each of the trip types and exports the data to a text file for viewing.
8. If the software attempts to reopen a database that has been either moved or deleted, the CarChip software will not crash.
9. If the software attempts to download more than 300 trips it will not crash.

What's new in version 2.1.0
1. Implemented Express & Full Walkthrough.
2. Assigns sequential Driver & Vehicle ID's starting from ID 1.
3. Automatically saves the document data without prompting for user intervention.
4. Improvements regarding display of home page and driver and vehicle summary pages.
5. Fixes problems with European version of Windows OS.
6. Fixes a bug that would improperly scale plots after canceling the plot setup screen.
7. Fixes a bug that would cause the program to crash if specific trouble codes were present in the downloaded data.

What's new in version 2.0.1
1. USB Support.
2. Support for toggling CarChip Alarm on/off (product #8225, #8245).
3. Support for toggling CarChip VIN mode on/off (product #8211, #8221, #8225, #8241, #8245).

This software update requires purchase of a CarChip, CarChipE/X, or CarChip Fleet (and Fleet Management Software), a PC running Windows 98SE/2000/ME/XP and a serial or USB port to connect to the CarChip.

Carchip SDK v1.0.6

The CarChip SDK is a development toolkit used by developers to interface with the CarChip device. All the SDK functions are encapsulated in the CarChipSDK.dll and can be accessed through the use of the header files "Structures.h" and "SDK.h" for C developers and the module "CarChipSDK.bas" for VB developers. The SDK kit acts as a high level tool and makes it much easier for developers to interface with a CarChip device.

What's new in version 1.0.6
CarChip.DLL version 1.7
DavisHid.DLL version 1.0.0.4 - USB Driver (Support for communicating with USB / CAN CarChips)
1. Handles the ReadinessCode introduced with firmware version 2.20.
2. Added GetLastErrorCodeStringLength() function that helps to allocate the correct buffer size needed by the calling routine for further processing.
3. Added GetSDKVersionComment(...) to obtain detailed information about the current version.

What's new in version 1.0.4
CarChip.DLL version 1.6.0.3
DavisHid.DLL version 1.0.0.4 - USB Driver (Support for communicating with USB CarChip's)
1. Changed function names from "EnableAlarm" and "IsAlarmEnabled" to "EnableBeep" and IsBeepEnabled".
2. Fixed crash that can occur if a bad/or corrupt time is retrieved on the CarChip.
3. Enhanced Accident log sorting routine. Fixed duplicate accident record problem.
4. Fixed memory crash after download.
5. Enhanced the robustness of the SDK. Properly handles memory corruption problems that can happen in the CarChip.
6. Avoid possible data corruption during download transfers. The DLL will re-download a page in memory if it finds any corruptions within that page.
7. Fixed ClearAccidentLogs() bug.
