Download Fleet DriveRight v3.9.5 Update
(size: 34.6 MB)

Note: This is only for users with Fleet version 3.0.0 or higher.

What's New in 3.9.5

  1. Added support for Athena SmartCard DriveRight download.
  2. Integrated Trip Map, Day Map, and Accident Log Map with Google Maps™ API V3.
  3. Fixed Accident Log Report date and time format in Report Viewer and Excel® to match the user's preference.

What's New in 3.9.4

  1. After performing a Backup, it now deletes the old records.
  2. Added Time in Motion to Days table.
  3. Fixed data overflow error in Vehicle Activity Report in Excel.
  4. Support for all CarChip parameters has been added to this release. Parameter logs are saved in CarChipParameters table and can be browsed from Database menu.
  5. Wireless device serial number is now shown on the Add New Base Station and Add New On-Board Module dialogs.
  6. Fixed Max Download Days error and type mismatch problem in Driver Safety Score report.
  7. Solved Google API key issue.
  8. Export/Import of CarChip table is now supported.
  9. Super User can now edit data in any location.
  10. MySQL ODBC Connector 3.51.29 is now included in setup program. The ODBC driver will be available on FMS CD and will be installed in DriveRight\MySQL Connector folder on the target machine.
  11. This release was built with CarChip.dll 3.2.2.0.

What's New in 3.9.3

Fleet DriveRight v3.8.0 Update

Note: This is only for users with Fleet version 3.0.0 or higher.

What's New in 3.8.0

2. GPS feature is implemented for CarChip Fleet Pro.
3. Wireless Download feature is extended to CarChip Fleet Pro.
4. The CarChip Alarm Volume feature is implemented.
5. Option to download all data in a CarChip or download only new data since last successful download, is implemented.
6. Speed Sample logging setting in Download Options is improved to give user more flexibility.
7. FMS is improved to handle trouble codes recorded during trip more accurately.
8. The access control to Corporate Structure now can be configured when add or edit user's privileges.

What's New in 3.7.2 (June 2007)
1. PIN code transferred by SmartCard to DriveRight will not be mistakenly set to 0 when FMS accesses the DriveRight settings screen.
2. Duplicate trip records will not cause a download failure when using a non-English database.

What's New in 3.7.1 (April 2007)
1. Password encryption feature is added to FMS to enforce security.
2. FMS now supports multiple super users. User type can be changed by a Super User through Browse/Add Users Dialog.
3. Driver Safety Score calculation is corrected when the unit mode is set to Kilometer and KM/hr.
4. Database error handling improved during data downloads. User will receive a failure notification if database error occurs.
5. CarChip settings and their default values are enforced based on CarChip models and firmware version.
6. Database CarChip record is synchronized with the CarChip device after user changes the Alarm and LED state.
7. ACS SmartCard reader is the only card reader to be used with FMS for smart card data transfer.
8. ACS Proprietary Driver installation procedures and troubleshooting information are added to FMS Getting Started Guide and User’s Manual.
9. Wireless Devices table backup problem is fixed in SQLServer and Access Database.
10. Database configuration process is improved to enforce database security and avoid object ownership problem in MS SQLServer database.
11. Add Unit (mph or km/hr) support for Top Speed in Excessive Speed report.
12. Set date/time problem and date/time synchronization problem after download are solved for DriveRight500.

Note: This DriveRight update requires purchase of a DriveRight console, DriveRight software v3.0.0 or above, a PC running Windows 98SE/NT/2000/ME/XP/Vista and a serial port to connect to the DriveRight console.

Compatibility: mySQL and Oracle engines are not compatible with Windows Vista at this time.


Fleet DriveRight v3.7.0 Update

Note: This is only for users with Fleet version 3.0.0 or higher.

What's New in 3.7.0
1. Wireless download capability. When used in conjunction with Davis Wireless On-Board Module (#8129) and Base Station for Wireless Download (#8130) data can be downloaded wirelessly from the vehicle.
2. The user is again required to enter the 8-digit registration number when FMS is started for the first time.
3. Upgrading FMS will not require the user to redefine the database type.
4. Stored procedures for Oracle database are integrated into FMS.
5. Data validation is added to ensure CarChip and DriveRight IDs are unique.
6. CarChip table import and export problem is fixed.
7. Download-only access right is properly applied to SmartCard operations.
8. Pin Code and Driver Logout Time now can be transferred to DriveRight using SmartCard.
9. Current location, displayed in the application title bar, is updated according user's modification.
10.GPS logging interval range is enforced to ensure accuracy of GPS logging hours.
11. Driver Logout Time for DriveRight will now correctly save the user's input.
12. Vehicles report is revised to contain all vehicles of the selected type (Light or Heavy) regardless whether the vehicle has an exclusive driver or not.

What's New in 3.6.0
1. The release date was removed from the application title bar.
2. Software supports the new DriveRight 600E.
3. Redundant GPS data is not written in the database to eliminate the erroneous "GPS data lost" message.
4. Software now handles downloading data greater than 365 days old properly.
5. Download Score in Driver Safety Report reports as 100 instead of 0 if Max Download Days is not available (shown as "---").
6. Days missing bug and Time Over Speed calculation was fixed. The longest Time Over Speed for one day that can be saved without causing a missing day record depends on user's database type: SQLServer/Access: 09:06:07, Oracle: 27:46:39, MySQL: 18:12:15
7. Oracle database stored procedures have been added.
8. Manual database creation scripts for SQLServer and Oracle have been added.

What's New in 3.5.0
1. Implemented batch editing feature.
2. Developed stored procedures for SQL Server to reduce logging time. Database administrators will have to add the stored procedures for Oracle database. Both these documents are placed under DOCS directory.
3. Added new fields to the CarChip database table for logging vehicle speed parameter, alarm & LED state.
4. Software displays the current location on the tile bar.
5. Software now logs trips in the database that occur in the same minute interval.
6. Implemented OBD readiness & trouble codes, for CarChip support.
7. Can now download a SmartCard with only download permissions. Previously SmartCard menu items were disabled for download only permissions.
8. Added new feature under Download Options for logging Vehicle Speed data in GPS database when using CarChip.
9. Fixed Days Since Last Download report. Previous version doesn’t display download dates for CarChip devices.
10. Displays exact trip start & end points on the MapPoint control in the software.

Note: This DriveRight update requires purchase of a DriveRight console, DriveRight software v3.0.0 or above, a PC running Windows 98SE/NT/2000/ME/XP and a serial port to connect to the DriveRight console.
